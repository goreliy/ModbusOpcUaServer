# Фаза 7 — конфигуратор (egui/eframe; Tauri исключён заказчиком)

## Решения
- Бинарь `opc-modbus-config` (крейт gui), eframe (glow), rfd для нативных диалогов
  открытия/сохранения. Кириллица: встроенный шрифт egui (Ubuntu) её покрывает.
- Модель редактирования: НАПРЯМУЮ `gateway_config::schema::v1::ConfigV1` (все поля pub);
  сохранение через `ConfigFile::V1` (schema_version пишется сам), чтение через
  serde. Валидация — `gateway_config::validate::validate(&cfg)` вживую, панель
  ошибок/предупреждений снизу.
- Разделы: OPC UA сервер (вкл. пользователей + кнопка «захэшировать пароль» через
  opcua_gateway::hash_password), Общие/Логи, Группы опроса, Каналы → Устройства →
  таблица регистров (все поля схемы: тип данных, порядок слов/байт, формулы,
  deadband, retentive, writable...).
- **Пробное чтение (dry-run)**: кнопка у устройства — фоновый поток с
  current_thread tokio-рантаймом читает настроенные регистры через mb-proto
  Transport и показывает сырые значения/ошибку. Киллер-фича наладки.
- UI на русском.
